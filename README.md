## 🗄️ إعداد قاعدة البيانات (Supabase)

1. أنشئ مشروعًا جديدًا على [supabase.com](https://supabase.com)
2. اذهب إلى **SQL Editor** → **New Query**
3. انسخ محتوى ملف [`supabase_schema.sql`](./supabase_schema.sql) والصقه، ثم اضغط **Run**
4. تحقق من نجاح العملية عبر **Table Editor** — يجب أن يظهر جدول `reports` بالأعمدة التالية:

   | العمود | النوع | الوصف |
   |---|---|---|
   | `id` | uuid | معرف فريد (تلقائي) |
   | `report_id` | text | رقم التتبع الظاهر للمستخدم |
   | `issue_type` | text | نوع المشكلة |
   | `location` | text | المنطقة / البلدية |
   | `description` | text | وصف المشكلة |
   | `status` | text | الحالة (`new`, `resolved`, ...) |
   | `created_at` | timestamptz | تاريخ الإرسال (تلقائي) |

5. تأكد من تفعيل **Row Level Security (RLS)** ووجود سياستين: *Public can insert reports* و *Public can read reports* (السكريبت ينشئهما تلقائيًا).

## 🔑 ربط الموقع بمشروعك على Supabase

داخل `index.html`، عدّل هذين المتغيرين ليطابقا مشروعك (Settings → API):

```js
const SUPABASE_URL = 'https://YOUR-PROJECT-REF.supabase.co';
const SUPABASE_PUBLISHABLE_KEY = 'YOUR-PUBLISHABLE-OR-ANON-KEY';
```

> استخدم دائمًا مفتاح **anon / publishable** فقط في الواجهة الأمامية. لا تضع مفتاح **service_role** في أي ملف يصل إليه المتصفح — فهو يمنح صلاحيات كاملة على قاعدة البيانات.

## 🚀 النشر على GitHub Pages

1. **Settings → Pages**
2. Source: **Deploy from a branch**
3. Branch: `main`, Folder: `/ (root)` → **Save**
4. بعد دقيقة تقريبًا، سيصبح الموقع متاحًا على:
   `https://YOUR-USERNAME.github.io/YOUR-REPO-NAME/`
